# 💳 Credit Card Dashboard

## 📌 Project Overview
This project is an end-to-end Credit Card Analytics Dashboard built using **SQL and Power BI**.  
The dashboard provides insights into credit card transactions, customer behavior, revenue trends, and key performance indicators (KPIs).

The objective of this project is to analyze customer spending patterns, revenue generation, interest earned, and demographic segmentation to support data-driven business decisions.

---

## 📊 Dashboard Features

### 🔹 Credit Card Transaction Report
- Total Revenue
- Total Interest Earned
- Total Transaction Amount
- Total Transaction Count
- Revenue by Card Category (Gold, Silver, Blue, Platinum)
- Revenue by Expenditure Type
- Quarterly Revenue Analysis
- Revenue by Payment Mode (Swipe, Chip, Online)

### 🔹 Credit Card Customer Report
- Revenue by Age Group
- Revenue by Income Group
- Revenue by Education Level
- Revenue by Marital Status
- Revenue by State
- Revenue by Dependents
- Gender-based Revenue Analysis
- Customer Job-wise Revenue Insights

---

## 🛠 Tools & Technologies Used
- **PostgreSQL / SQL** – Data storage & querying
- **Power BI** – Data visualization & dashboard creation
- **CSV Files** – Raw data source
- **DAX** – KPI calculations and measures

---

## 🗂 Dataset Information
The project includes:
- `credit_card.csv` – Transaction data
- `customer.csv` – Customer demographic data
- Additional CSV files for data updates

---

## 📈 Key Insights
- Blue card category generates the highest revenue.
- High-income group customers contribute significantly to total revenue.
- Majority revenue comes from chip-based transactions.
- Married customers show higher spending patterns.
- Graduate customers contribute the highest revenue among education groups.

---

## 🚀 How to Use
1. Import CSV data into PostgreSQL.
2. Perform data cleaning and transformation using SQL.
3. Connect PostgreSQL to Power BI.
4. Load and build dashboard using Power BI visuals.
5. Apply filters for dynamic analysis.
